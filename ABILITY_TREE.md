# Дерево способностей — Комбинации характеристик

> **Этот файл — наш общий интерфейс.**
> Редактируй любую таблицу — я буду читать актуальную версию и подстраивать код под неё.

---

## 1. Как это работает

```
Игрок качает 4 характеристики → Комбинации ОТКРЫВАЮТ способности → Очки активации ВКЛЮЧАЮТ их
```

- **Открытие** — способность появляется в списке доступных (но ещё не работает)
- **Активация** — игрок тратит очко активации, чтобы поместить способность в слот
- **6 активных слотов** + **3 пассивных слота** (как сейчас в GameAbilityBar)

## 7. Правила (для кода)

1. **Открытие**: проверяем `S >= req.strength && A >= req.agility && I >= req.intellect && C >= req.charisma` **И** все `requiredAbilities` активированы в слотах
2. **Активация**: у токена хранится `activationPoints` — сколько доступно. При размещении в слот тратится 1 очко
3. **Слоты**: 6 активных + 3 пассивных (как сейчас). Нельзя поставить в слот способность, если нет очка активации
4. **Деактивация**: можно снять способность из слота и вернуть очко активации (перестроить билд). **Важно**: если снять пререквизит — все зависимые синергии автоматически деактивируются (каскад)
5. **Потеря требований**: если стат упал (проклятие, отравление) ниже требования — способность временно недоступна, но слот не очищается
6. **NPC**: для NPC с `autoLevel: true` способности назначаются автоматически по статам
7. **Синергии (Тир S)**: помимо требований к статам, проверяют `requiredAbilities` — массив ID способностей, которые **должны быть активированы** (в слоте). Это означает, что игрок платит слотами за доступ к мощным комбо
8. **Свитки (будущее)**: специальные предметы могут разблокировать способности, минуя часть требований (например, «Свиток крови» открывает blood_bolt без I≥3). Учитывается в поле `unlockedByScroll` у токена

## 8. Формат данных (будущая структура)

```js
// В abilities.js — к каждой способности добавится:
{
  ...existingFields,
  tier: 1,                    // 1-4, или 'S' для синергий
  requirements: {             // Минимальные статы для открытия
    strength: 0,
    agility: 3,
    intellect: 0,
    charisma: 0,
  },
  requiredAbilities: [],      // Для синергий: ID активированных способностей-пререквизитов
}

// В токене (placedTokens) — новые поля:
{
  activationPoints: 3,        // Сколько очков активации доступно
  unlockedAbilities: ['blood_bolt', 'quick_step', ...],  // Автовычисляемый массив
  unlockedByScroll: [],       // Способности, открытые свитками (игнорируют стат-требования)
}
```

### Баланс синергий

> **Цена синергии** = слоты на пререквизиты + слот на саму синергию.
> Например, `blood_gravity_arrow` требует 3 слота: `blood_bolt` + `gravity_bolt` + сама стрела.
> У игрока всего 6 активных + 3 пассивных слота, поэтому синергия — это **осознанная специализация**.
>
> **Ультимативные синергии** (`blood_vortex`, `gravity_storm`, `war_avatar`) требуют T3+ пререквизиты
> и I≥7 или S≥5+C≥5, что ограничивает их 13-15 уровнем.
